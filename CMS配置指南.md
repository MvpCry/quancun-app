# 去俺村 — 微信云开发内容管理(CMS)配置指南

## 前期准备

### 1. 部署云函数
在微信开发者工具中，右键以下云函数 → 上传并部署：云端安装依赖：
- `calcRouteDistance`（新建，需上传）
- `getAttractions`（已有，确认已部署）
- `getRoutes`（已有，确认已部署）
- `importDefaultData`（已有，确认已部署）

### 2. 确认数据库集合存在
云开发 → 数据库，确认以下集合已创建且有数据：
- `attractions` - 景点
- `routes` - 路线
- `reviews` - 评论

如果某个集合为空，去云函数面板调用 `importDefaultData`：
```json
{ "action": "import" }
```

---

## 一、开通内容管理(CMS)

**微信开发者工具** → 点击顶部「云开发」→ 左侧菜单找到「内容管理」→ 点击开通。

如果没有「内容管理」入口（新版已改名）：
- 访问 https://console.cloud.tencent.com/tcb
- 选择你的环境 → 左侧「内容管理」→ 开通
- 或使用「微搭低代码」→ 快速搭建

开通后会得到一个管理后台网址，微信扫码即可登录。

## 二、配置景点管理 (attractions)

在内容管理中创建内容模型「景点」：

| 字段名 | 显示名 | 类型 | 必填 | 说明 |
|--------|--------|------|------|------|
| name | 景点名称 | 文本 | ✅ | 如：王林坡村 |
| address | 详细地址 | 文本 | ✅ | 如：山东省泰安市泰山区邱家店镇王林坡村 |
| category | 分类 | 枚举 | ✅ | 选项：rural(乡村)/red(红游)/family(亲子)/culture(文化) |
| tags | 标签 | 数组(文本) | | 如：["美丽乡村","农家体验"] |
| images | 图片 | 数组(图片) | | 上传到云存储 |
| introduction | 简介 | 富文本 | | 景点详细介绍 |
| description | 摘要 | 文本 | | 简短描述(列表页展示) |
| location | 坐标 | 对象 | | `{"latitude":36.xxx,"longitude":117.xxx}` |
| openTime | 开放时间 | 文本 | | 如：9:00-17:00 / 全天 |
| ticketPrice | 门票 | 数字 | | 0=免费 |
| rating | 评分 | 数字 | | 系统自动计算，初始填4.5 |
| reviewCount | 评论数 | 数字 | | 系统自动计算，初始填0 |

> **坐标获取方法**：打开 https://lbs.qq.com/getPoint/ → 搜索村名 → 复制 GCJ-02 坐标填入 location 字段。

## 三、配置路线管理 (routes) — 核心

在内容管理中创建内容模型「路线」：

| 字段名 | 显示名 | 类型 | 必填 | 说明 |
|--------|--------|------|------|------|
| name | 路线名称 | 文本 | ✅ | 如：泰安乡村一日游 |
| description | 路线简介 | 富文本 | ✅ | 路线详细介绍 |
| coverImage | 封面图 | 图片 | ✅ | 列表展示用 |
| tags | 分类标签 | 数组(枚举) | ✅ | 多选：一日游/两日游/乡村游/亲子游/红色旅游/文化旅游 |
| attractions | 包含景点 | 关联(多选) | ✅ | **关联到 attractions 集合，按选择顺序排列** |
| totalDistance | 总里程(km) | 数字 | | 系统自动计算，不用填 |
| estimatedTime | 预估时长(h) | 数字 | | 系统自动计算，不用填 |
| attractionCount | 景点数 | 数字 | | 系统自动计算，不用填 |

### 关键：设置关联字段

attractions 字段配置为「关联」类型：
1. 关联集合：`attractions`
2. 多选：开启
3. 显示字段：`name`
4. 排序：按选择顺序

### 关键：设置云函数触发器

在路线模型的「触发器」设置中：
1. **afterCreate** → 选择云函数 `calcRouteDistance`
2. **afterUpdate** → 选择云函数 `calcRouteDistance`

这样每次新增或编辑路线，系统会自动计算总里程和预估时长。

## 四、操作示例

### 新增景点
1. 打开 CMS 后台 → 景点管理 → 新增
2. 填写村名：`王林坡村`
3. 上传图片（点图片字段上传）
4. 填写地址、选择分类、打标签
5. 填入坐标：`{"latitude":36.2007,"longitude":117.2413}`
6. 保存 → 前端首页自动显示

### 编排路线
1. 打开 CMS 后台 → 路线管理 → 新增
2. 填写路线名称：`泰安乡村两日游`
3. 上传封面图
4. **在「包含景点」关联字段中勾选**：白马石村 → 王林坡村 → 朱家洼村（勾选顺序=游玩顺序）
5. 选择标签：勾选 `乡村游` + `两日游`
6. 填写简介
7. 保存 → 云函数自动计算里程和时长 → 前端分类入口自动展示

## 五、标签绑定规则

CMS 中路线 `tags` 字段的值决定首页分类展示：

| 路线 tags | 首页入口 |
|-----------|----------|
| 包含 `乡村游` | 点击「乡村游」图标 → 列表展示 |
| 包含 `亲子游` | 点击「亲子游」图标 → 列表展示 |
| 包含 `红色旅游` | 点击「红游」图标 → 列表展示 |
| 包含 `文化旅游` | 点击「文化游」图标 → 列表展示 |
| 包含 `一日游` | 路线列表「一日游」筛选可见 |

> 一条路线可以同时勾选多个标签（如 `乡村游` + `亲子游`），同时出现在多个分类下。

## 六、首次导入

如果数据库尚未有景点和路线，调一次 `importDefaultData` 云函数：

```json
{ "action": "import" }
```

导入后 CMS 中会自动出现 3 个村 + 1 条路线，可直接在此基础上编辑。
